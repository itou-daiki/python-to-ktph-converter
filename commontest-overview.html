📚 共通テスト用プログラム表記とは

共通テスト用プログラム表記は、大学入学共通テストの「情報」科目で使用される擬似言語です。

  • 日本語ベース：日本語のキーワードを使用
  • 教育用設計：プログラミング的思考の理解を重視
  • 言語非依存：特定のプログラミング言語に依存しない
  • 視覚的理解：アルゴリズムの構造が分かりやすい

📋 共通テスト表記の特有記法

🔤 制御記号

ブロック構造を示すために特別な記号を使用します。

もし 条件 ならば:
｜  処理1
｜  処理2
⌐  処理3  // ブロックの最後

｜：ブロック内の処理
⌐：ブロックの最後の処理

🧮 共通テスト表記でのアルゴリズム例

線形探索

データ = [2, 3, 4, 10, 40]
探索値 = 【外部からの入力】
見つかった = False

i を 0 から 要素数(データ)-1 まで 1 ずつ増やしながら繰り返す:
｜  もし データ[i] == 探索値 ならば:
｜  ｜  表示する("値 " + 探索値 + " は位置 " + i + " にあります")
｜  ｜  見つかった = True
⌐  ｜  繰り返しを抜ける

もし 見つかった == False ならば:
｜  表示する("値 " + 探索値 + " は見つかりませんでした")

バブルソート

データ = [64, 34, 25, 12, 22, 11, 90]
n = 要素数(データ)

表示する("ソート前のデータ:")
i を 0 から n-1 まで 1 ずつ増やしながら繰り返す:
｜  表示する(データ[i])

// バブルソート実行
i を 0 から n-1 まで 1 ずつ増やしながら繰り返す:
｜  j を 0 から n-i-2 まで 1 ずつ増やしながら繰り返す:
｜  ｜  もし データ[j] > データ[j+1] ならば:
｜  ｜  ｜  // 要素を交換
｜  ｜  ｜  一時 = データ[j]
｜  ｜  ｜  データ[j] = データ[j+1]
⌐  ｜  ⌐  データ[j+1] = 一時

表示する("ソート後のデータ:")
i を 0 から n-1 まで 1 ずつ増やしながら繰り返す:
｜  表示する(データ[i])

二分探索

データ = [3, 18, 29, 33, 48, 52, 62, 77, 89, 97]
個数 = 要素数(データ)
表示する("0～99の数字を入力してください")
値 = 【外部からの入力】
左 = 0
右 = 個数 - 1
終了 = False

左 <= 右 and 終了 == False の間繰り返す:
｜  中央 = (左 + 右) ÷ 2
｜  もし データ[中央] == 値 ならば:
｜  ｜  表示する(値 + "は" + 中央 + "番目にありました")
｜  ｜  終了 = True
｜  そうでなくもし データ[中央] < 値 ならば:
｜  ｜  左 = 中央 + 1
｜  そうでなければ:
⌐  ｜  右 = 中央 - 1

もし 終了 == False ならば:
｜  表示する(値 + "は見つかりませんでした")

💡 共通テスト表記学習のポイント

1. 日本語キーワードに慣れる

英語ベースのプログラミング言語とは異なり、日本語のキーワードを使用します。

  • 「もし～ならば」「そうでなければ」
  • 「～の間繰り返す」「～ずつ増やしながら」
  • 「表示する」「【外部からの入力】」

2. 制御記号を理解する

ブロック構造を示す記号の使い方を覚えましょう。

  • ｜：ブロック内の処理を示す
  • ⌐：ブロックの最後の処理を示す
  • インデント（字下げ）で階層を表現

3. アルゴリズムの流れを重視

構文よりもアルゴリズムの論理的な流れを理解することが重要です。

  • 条件分岐の使い分け
  • 繰り返し処理の制御
  • 変数の役割と使い方

4. 実用的なアルゴリズムで練習

基本的なアルゴリズムを共通テスト表記で実装してみましょう。

  • 探索アルゴリズム（線形探索、二分探索）
  • ソートアルゴリズム（バブルソート、選択ソート）
  • 数学的計算（最大公約数、素数判定）
